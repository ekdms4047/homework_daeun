태그 종류

문단태그 p

- 블록 요소이나 문단 태그는 또 다른 블록 요소를 포함하지 않음. 주로 인라인 요소에만 넣음
- 문단을 정의하는 요소
- 문단은 하나 이상의 문장으로 구성되며 이야기의 독립적인 단위를 형성. 문단의 시작은 새로운 줄로 나타냄
- 태그 구성: 요소Element 속성Attribute = 값Value
- 시맨틱 마크업 Sementic Markup
- 태그가 가진 의미에 집중하여 컨텐츠 자체에 의미를 부여하는 것. 웹 접근성 부분에서도 중요

일반 강조 태그 em

- 인라인 요소
- 일반적인 강조의 의미로 사용
- 기울여져서 표현됨
- 디자인을 위해 em을 사용하면 안됨
- 비슷한 태그로 i가 있음. i는 단순 구분을 위해 사용함

강한 강조 태그 strong

- 인라인 요소

제목 태그 h

- 인라인 요소로 중첩 작성 가능
- h1 : 문서의 메인 제목. 한 문서에 한번만 작성 권장
- h2 ~ h6 : 제목 태그로 한 문서안에 여러번 사용 가능
의미없이 제목 단계를 건너띄지 않음

사용자 정보 태그 address

- 블록 요소
- 연락처, url, 주소, sns, 사업자명, 사업자 번호 등

작품의 제목 태그 cite, 인용 태그 blockquote

- 블록 요소
cite : 기울여짐, cite blockquote : 들여쓰기 됨, 출처 주소 기입

줄바꿈 태그 br

- 인위적인 여백을 만들기 위해 사용하지 않으며 반드시 줄바꿈이 필요한 경우에만 사용
- 반응형 웹에서는 br 태그를 이용해 줄바꿈 하기보단 span태그를 이용해 줄바꿈을 하는편

콘텐츠 분할 span

- 가장 대표적인 인라인 요소
- 인라인으로 구조를 나누기 위한 태그
- 브라우저상의 특징이 없음
- 주로 다른 텍스트와 구분하고 싶지만 적절한 태그가 없을때 사용
- id,class 속성과 함께 사용

콘텐츠 분할 div

- 가장 대표적인 블록 요소
- 구조를 나누기 위한 태그로 브라우저상의 특징이 없음
- 단순히 레이아웃을 구성하기 위해 사용
- id, class 속성과 함께 사용

순서 있는 목록 태그 ol / 순서 없는 목록 태그 ul

- 블록 요소 / 자동으로 숫자가 부여됨
- ol(ul)과 li 사이엔 다른 태그가 올수 없음
li 태그 안에는 다양한 태그 작성 가능
- ol(ul)과li는 함께 사용. 둘 중 하나만 사용할 수 없음
- 꼭 순서가 필요한 리스트에 작성

정의형 목록 태그 dl

- dl,dd,dt 전부 블록 요소
- 용어의 정의,설명,참고문헌,메타데이터 등 다양하게 사용 가능
- 일부 os에서 dl의 정의가 다르기 때문에 접근성이 좋지 않음
- dt: 항목의 제목, 용어 등
- dd: 위의 제목이나 용어의 설명을 작성하는 부분
- dd: 들여쓰기 됨

앵커 anchor 태그 a

- 인라인 요소
- href 속성을 이용하여 하이퍼 텍스트 설정
- href 속성을 통해 다른 페이지나 같은 페이지의 어느위치, 파일, 이메일 주소와 그 외 다른 url로 연결할 수 있는 하이퍼링크를 만듦
- 기본 색상 파랑, 방문하면 보라색, 마우스로 누르고 있으면 빨강
- 링크를 의미하는 밑줄로 표기
- a의 속성 값을""로 비워두지 않음 href="#"처럼 임시링크 기호 대체 표기
- target="_blank"새창으로 열기 속성

이미지 태그 img

- 인라인 요소
- 빈 요소, 빈 태그로 종료 태그 없음
- src="이미지 경로" alt="이미지 대체 텍스트"
- alt의 경우 웹표준의 기본이므로 꼭 작성
- 컨텐츠로 분류되고 닫아주는 태그가 없음에도 범위를 가지며 가로 세로 값 조절 가능
